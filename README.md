# 📍 地址批量轉經緯度工具

利用 Google Maps Geocoding API，透過圖形化介面（GUI）將 Excel / CSV 檔案中的地址批量轉換為經緯度座標。

## 功能特色

- 🖥️ **圖形化操作介面** — 免打指令，點選即用
- 📂 **檔案瀏覽對話框** — 直接選取 Excel (.xlsx/.xls) 或 CSV 檔案
- 📋 **動態欄位選擇** — 自動讀取檔案標題，下拉選單選擇地址欄位
- 📊 **即時進度顯示** — 進度條 + 成功/失敗計數
- 📝 **彩色執行記錄** — 即時顯示每筆轉換結果（綠色成功、紅色失敗、橘色警告）
- 🔄 **API 超限自動重試** — 指數退避機制（最多 3 次）
- ⏹️ **可隨時取消** — 中斷長時間作業

## 前置需求

- Python 3.10+
- Google Maps Geocoding API 金鑰（[申請方式](https://developers.google.com/maps/documentation/geocoding/get-api-key)）

## 安裝

```bash
pip install pandas requests openpyxl
```

## 使用方式

```bash
python geocode_addresses.py
```

啟動後會開啟圖形化介面，操作步驟如下：

### 步驟一：選擇檔案

點擊「**瀏覽…**」按鈕，選取包含地址資料的 Excel 或 CSV 檔案。

### 步驟二：設定

- **地址欄位**：從下拉選單中選擇包含地址的欄位（程式會自動偵測常見名稱如 `ADDR`、`地址`）
- **API 金鑰**：輸入你的 Google Maps Geocoding API 金鑰

### 步驟三：開始轉換

點擊「**▶ 開始轉換**」，選擇輸出檔案的儲存位置後即開始執行。

## 輸入格式

Excel 或 CSV 檔案需包含一個地址欄位，例如：

| 編號 | ADDR |
|------|------|
| 1 | 新北市板橋區中山路一段161號 |
| 2 | 新北市新莊區中正路1號 |

## 輸出格式

原始資料加上三個新欄位：

| 編號 | ADDR | 經度 | 緯度 | 錯誤原因 |
|------|------|------|------|----------|
| 1 | 新北市板橋區中山路一段161號 | 121.4591 | 25.0118 | |
| 2 | 無效地址 | | | ZERO_RESULTS: 無結果 |

## 注意事項

- Google Maps Geocoding API 為**付費服務**，請留意用量
- 每次 API 請求間隔 50ms，避免觸發頻率限制
- 超過查詢限制時會自動以指數退避重試（最多 3 次）
- 支援 Excel (.xlsx/.xls) 和 CSV (.csv) 格式的輸入與輸出
